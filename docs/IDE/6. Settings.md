# ⚙️ Project Settings

## Overview
The **Settings** tab is where you configure how your project runs. This includes the startup command, dependencies, and environment variables.

<img src="/img/IDE/12.png" alt="Project Settings Overview" width="100%">

## Startup Configuration
The startup command is crucial for running your application correctly. It tells the server what command to execute when you click "Start".

<img src="/img/IDE/13.png" alt="Startup Command Configuration" width="100%">

### Environment Variables
Depending on the programming language or framework you selected when creating the server (Node.js, Python, etc.), the available variables will vary.

**Common Variables:**
- **Git Repo Address**: If you want to pull code from a GitHub repository automatically.
- **Install Branch**: The specific branch to pull (e.g., `main` or `master`).
- **User Uploaded Files**: Toggle to allow or skip installation steps if you are uploading files manually.
- **Auto Update**: Configure whether to pull the latest changes on startup.

**Language Specifics:**
- **Node.js**: You might see options for `MAIN_FILE` (e.g., `index.js`), `NODE_PACKAGES` (for additional global packages), or `PACKAGE_MANAGER` (npm/yarn).
- **Python**: You might see options for `MAIN_FILE` (e.g., `app.py`) or `REQUIREMENTS_FILE` (e.g., `requirements.txt`).

<img src="/img/IDE/14.png" alt="Environment Variables Detail" width="100%">

## Danger Zone
Just like with Game Servers, this section allows you to **Reinstall** your project.
- **Reinstall Server**: This will wipe all your current files and reset the environment to its initial state. **Use with caution.**
